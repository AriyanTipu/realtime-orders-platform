# Security policy

This is a portfolio/demonstration system. It ships with deliberately
insecure development defaults (documented in `.env.example`) and a
DEMO_MODE that exposes unauthenticated demo endpoints; both must be
disabled for any non-local deployment, as the deployment runbook states.

If you find a vulnerability, email **ariyantipu39@gmail.com** with details
and reproduction steps. Please do not open a public issue for security
reports.
